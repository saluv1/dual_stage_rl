## Phase I
The main purpose of Phase I of PS2-RL is to apply the concept of Backup Control Barrier Functions and the Safe Arrival Policy to train the backup policy, which will then be used to derive the implicit control-invariant set.

## Phase II
The main purpose of Phase II of PS2-RL is to make constrained policy by mixing the concept of SAC-RL and Hard-net concept (which is implemented by qpax) and A<sub>BCBF</sub>, b<sub>BCBF</sub> will be derived by Safe Arrival Policy which generates control-invariant set implicitly.
it should solve optimization problem via qpax (qp solver) in jax
  <br/>so, I bring three main codes that will be combined into one architecture. 
  <br/> [SAC in jax](https://github.com/aliciafmachado/sac) 
  <br/>[hardnet in pytorch](https://github.com/azizanlab/hardnet) 
  <br/>[making trajectory via BCBF idea](https://github.com/davidvwijk/DR-bCBF) 
  <br/>[powerloop reference trajectory from acrobatic drone](https://github.com/uzh-rpg/deep_drone_acrobatics/blob/master/acrobatic_trajectory_helper/src/circle_trajectory_helper.cpp)
Progress
- Dynamics (engine: rk4, visualization: mujoco)
- ps2_sac (complete, but it is basically generated by gpt5.5, so I will proofread all the code.)
